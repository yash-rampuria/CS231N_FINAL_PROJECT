# CS 231N Project: Goal-Conditioned Visuomotor Navigation for Quadrotors

## One-Line Summary
A deep learning model that maps raw RGB video sequences + a goal-heading vector directly to quadrotor velocity commands — no maps, no privileged state, no depth sensors.

## Core Problem
In unstructured environments (forests, collapsed buildings), pre-computed maps are unavailable. The question we are answering: **what visual representations and temporal architectures are sufficient for goal-directed, obstacle-avoiding flight?**

## What We Are Building

**Input:** A window of RGB frames `I_{t-k:t}` + a 3D goal-heading unit vector `g` + a scalar distance-to-goal `d`  
**Output:** Reference velocity commands `(vx, vy, vz, ψ̇)`  
**Low-level control:** Decoupled — an inner-loop geometric controller handles thrust conversion, so the neural net only needs to learn high-level navigation.

## Architecture
- **Backbone:** 3D-ResNet18 (spatiotemporal) or Depth Anything V2
- **Training:** Behavior cloning via MSE loss against expert velocity commands
- **Optional:** DAgger to reduce compounding errors from distribution shift

> **Note:** Specific model and architecture choices (backbone, loss formulation, output head) are subject to change as project decisions evolve. Treat the above as the current working plan, not a fixed spec.

## Data & Simulation
- Expert data generated by an NMPC / geometric controller + RRT path planner
- Simulators: **AirSim** or **FiGS** (photorealistic 3D Gaussian Splatting sim, 130 fps, from Stanford MSL — same lab)
- Expert tuples: `(RGB frame window, goal heading, reference velocity commands)`

## Training Paradigm: Student-Teacher Imitation Learning
A geometric controller acts as the **teacher/expert**. The neural network is the **student**, learning to replicate the expert's velocity commands from visual input alone.

## Output Formulations (simplest to most complex)
1. Fixed-altitude 2D navigation (simplest baseline)
2. Next-waypoint prediction
3. Long-horizon trajectory prediction

## Evaluation Metrics
- Collision-free waypoint success rate (closed-loop simulation trials)
- Velocity command RMSE vs. expert
- Grad-CAM visualizations to verify attention aligns with obstacles

## Key Design Decisions
- No explicit map representation at any stage
- Vision backbone must implicitly encode geometry/obstacle proximity from temporal RGB
- Behavior cloning first; DAgger as a stretch goal to handle distribution shift

## Related Work Context
- Loquercio 2021: end-to-end visual policies work but are distribution-shift sensitive
- Ma 2019: intermediate latent representations stabilize RGB policy learning
- SOUS VIDE (Low et al. 2024, Stanford MSL): zero-shot sim-to-real via 3DGS training — closely related, FiGS is from this paper
- Xue & Gonsalves 2021: VAE + SAC for vision-to-control obstacle avoidance
